## Introduction

### Role Overview

The Dean is the senior academic head responsible for academic leadership, faculty oversight, department performance, curriculum quality, accreditation readiness, and institutional excellence within their faculty/school. This role provides high-level academic, administrative, and operational insights while maintaining appropriate access boundaries.

### Key Responsibilities

- **Academic Leadership**: Oversee all academic operations and department performance
- **Faculty Oversight**: Monitor faculty workload, performance, and development
- **Curriculum Management**: Approve syllabus submissions and monitor curriculum quality
- **Approvals & Workflow**: Process requests from HODs and departments
- **Accreditation**: Monitor and maintain accreditation readiness (NAAC/NBA)
- **Compliance**: Ensure academic compliance and quality standards
- **Communication**: Coordinate with departments, faculty, and staff
- **Reporting**: Generate comprehensive academic reports and insights

### Access Requirements

- **Role**: Dean (Role ID: 30)
- **Permissions**: 224+ specific permissions for dean operations
- **Multi-Tenancy**: Access limited to your assigned school only

## Approvals & Workflow

### All Approvals Page

**Location**: `/dashboard/dean/approvals`

#### Accessing Approvals

1. Navigate to the **Approvals** tab on the main dashboard
2. Click **"All Approvals"** card or navigate directly to `/dashboard/dean/approvals`
3. View all approval requests

#### Approval Types

- **HOD Requests**: Requests from Heads of Departments
- **Syllabus Submissions**: Syllabus approval requests
- **Training Nominations**: Training and development requests
- **Class Reschedule**: Class rescheduling requests
- **Exam Changes**: Examination modification requests
- **Staff Leave**: Leave escalation requests

#### Viewing Approval Requests

1. View the approvals table showing:
   - Request type
   - Title
   - Requester
   - Department
   - Priority (High/Medium/Low)
   - Status (Pending/Approved/Rejected)

#### Filtering Approvals

- **By Type**: Filter by request type
- **By Status**: Filter by approval status
- **By Priority**: Filter by priority level
- **By Search**: Search by title, requester, or department

#### Approving Requests

1. Click the **Approve** button (green checkmark) on a pending request
2. Optionally add approval comments
3. Click **Approve** to confirm
4. The request status will update to "Approved"

#### Rejecting Requests

1. Click the **Reject** button (red X) on a pending request
2. Enter a rejection reason (required)
3. Click **Reject** to confirm
4. The request status will update to "Rejected"

#### Viewing Request Details

1. Click the **View** button (eye icon) on any request
2. View complete request information:
   - Request type and title
   - Requester details
   - Department information
   - Priority and status
   - Submission date
   - Description and details

#### Approval Statistics

The approvals page displays:
- **Pending Approvals**: Count of pending requests

- **Rejected**: Count of rejected requests
- **Total Requests**: Total number of requests

## Security & Compliance

### Access Control

#### Role-Based Access

- Dean role has access to all dean-specific functions
- Access is restricted to assigned school only
- No cross-school data access is permitted

#### Permission Requirements

All functions require specific permissions:
- **Academic Oversight**: `academic:read`, `department:read`
- **Faculty Management**: `staff:read`, `faculty:read`
- **Approvals**: `approval:hod:approve`, `workflow:manage`
- **Reports**: `report:read`, `report:academic:read`
- **Compliance**: `compliance:read`, `accreditation:readiness:read`

### Data Security

#### Multi-Tenancy

- All data is filtered by `school_id` or `school_code`
- No access to other schools' data
- School boundaries are strictly enforced

#### Sensitive Data Masking

- Sensitive student information may be masked where appropriate
- Personal identifiers are protected
- Academic data is accessible only to authorized roles

## Troubleshooting

### Common Issues

#### Cannot Access Dashboard

**Problem**: Unable to access the Dean dashboard

**Solutions**:
1. Verify you have the "dean" role assigned
2. Check that you have required permissions
3. Ensure you're accessing the correct school code
4. Clear browser cache and cookies
5. Contact system administrator

#### Approval Actions Not Working

**Problem**: Cannot approve or reject requests

**Solutions**:
1. Verify you have `approval:hod:approve` permission
2. Check that the request is in "Pending" status
3. Ensure you're connected to the internet
4. Refresh the page and try again
5. Check browser console for errors

#### Reports Not Loading

**Problem**: Reports page shows no data

**Solutions**:
1. Verify you have `report:read` permission
2. Check the selected time period
3. Ensure data exists for the selected period
4. Refresh the page
5. Try selecting a different time period

#### Communication Not Sending

**Problem**: Emails/SMS not being sent

**Solutions**:
1. Verify you have `email:send` or `sms:send` permission
2. Check recipient email addresses/phone numbers
3. Verify email/SMS service is configured
4. Check communication history for delivery status
5. Contact IT support if issue persists

### Error Messages

#### "Insufficient Permissions"

**Cause**: You don't have the required permission for the action

**Solution**: Contact system administrator to assign the required permission

#### "School Access Denied"

**Cause**: You're trying to access data from a different school

**Solution**: Ensure you're using the correct school code in the URL

#### "Rate Limit Exceeded"

**Cause**: Too many requests in a short time

**Solution**: Wait a few minutes and try again

## FAQs

### General Questions

**Q: How do I access the Dean dashboard?**  
A: Navigate to `/[school-code]/dashboard/dean` after logging in with your dean credentials.

**Q: What permissions do I need?**  
A: You need the "dean" role and 224+ specific permissions defined in the RBAC system.

**Q: Can I access data from other schools?**  
A: No. The system enforces multi-tenancy, and you can only access data from your assigned school.

**Q: How do I approve a request?**  
A: Navigate to the Approvals page, find the request, click the Approve button, optionally add comments, and confirm.

**Q: How do I view department performance?**  
A: Navigate to the Academic tab on the main dashboard to view department performance metrics.

### Approval Questions

**Q: What types of requests can I approve?**  
A: You can approve HOD requests, syllabus submissions, training nominations, class reschedules, exam changes, and staff leave escalations.

**Q: Can I reject a request without providing a reason?**  
A: No. A rejection reason is required when rejecting any request.

**Q: How do I view approval history?**  
A: Navigate to the Approvals page and use the status filter to view approved or rejected requests.

### Syllabus Questions

**Q: How do I approve a syllabus?**  
A: Navigate to Syllabus Approval page, find the submission, click Approve, add optional comments, and confirm.

**Q: Can I download syllabus files?**  
A: Yes. Click the Download button on any syllabus submission to download the PDF file.

**Q: What happens after I approve a syllabus?**  
A: The syllabus status changes to "Approved" and the department is notified.

### Training Questions

**Q: How do I approve a training nomination?**  
A: Navigate to Training Nominations page, find the nomination, click Approve, review details, add optional comments, and confirm.

**Q: Can I see the estimated cost of training?**  
A: Yes. The training nomination details include estimated cost information.

**Q: What information do I need to approve training?**  
A: Review the training name, provider, date, duration, estimated cost, and justification before approving.

### Communication Questions

**Q: How do I send an email to all faculty?**  
A: Navigate to Communications, select Bulk Messaging, choose "All Faculty", compose message, and send.

**Q: Can I use email templates?**  
A: Yes. The system provides pre-configured templates for common communications like evaluation reminders and meeting invitations.
